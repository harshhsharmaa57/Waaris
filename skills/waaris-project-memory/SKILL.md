# Waaris Project Memory

1. Read `.context.md` before beginning work and read `README.md` completely before implementation changes.
2. Check `TASKS.md`, `PROGRESS.md`, `PLAN.md`, and `DECISIONS.md` to avoid duplicating or bypassing planned work.
3. Work only within the active phase; explicitly enforce the advanced-cryptography/blockchain/DP gate.
4. Before finishing, update `.context.md`, `PROGRESS.md`, and `TASKS.md`, plus every affected living specification.
5. Validate the change and create one focused commit with an accurate message; do not absorb unrelated worktree changes.
